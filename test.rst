测试流程规范
=============
前言
---------------
- **测试流程千万条，规范操作第一条。测试流程不规范，开发人员两行泪**

测试开发周流程
---------------
- 1.1 产品经理对提出的需求进行解读，开发人员与测试人员进行评审确认以及细节确认
- 1.2 新功能case编写，测试case编写
- 1.3 case评审（case评审由开发产品测试共同参加）

测试周与验收周
---------------
- 2.1 新功能大流程测试：新功能测试大流程能走通
- 2.2 新功能细节测试 ：新功能各个影响点测试
- 2.3 新功能回归测试：回归所有问题点无较大影响后，回归测试
